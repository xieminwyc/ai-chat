# 从前端到全栈 AI Agent 的路线图

## 这份文档是干什么的

这不是一个“继续打磨聊天页面”的待办，而是这个项目下一阶段的主线规划。

目标只有一个：

**把当前的流式聊天全栈应用，继续演进成一个最小但真实的 AI agent 产品。**

---

## 当前阶段已经具备的基础

你已经不是从零开始了。当前项目已经有这些能力：

- `Next.js App Router`
- `route.ts` 服务端接口
- 前端 `fetch -> response.body.getReader()` 的流式读取
- `PostgreSQL + Prisma` 聊天持久化
- 第三方模型接入
- 基础测试、lint、类型检查

这意味着下一阶段不该继续困在“怎么做一个聊天页面”，而是开始进入：

- agent 工具调用
- 任务编排
- 运行日志
- 多用户和部署

---

## 下一阶段最重要的学习主线

### 1. Tool Calling / Agent Loop

这是从“AI chat”变成“AI agent”的第一步。

你要真正学会的不是多接一个模型，而是这条链路：

1. 用户发起请求
2. 模型先判断要不要调用工具
3. 服务端执行工具
4. 把工具结果喂回模型
5. 模型生成最终回答
6. 全过程落库

建议先做一个最小工具：

- `get_time`
- 或 `web_search`

推荐顺序：

1. 先做 `get_time`
2. 再做 `web_search`

原因：

- `get_time` 最容易帮助你理解 agent loop
- `web_search` 更接近真实产品，但复杂度更高

### 2. Observability / Run Trace

做 agent 以后，最怕的是“它为什么这么做你根本看不见”。

所以第二条主线要学的是：

- 每次 agent run 的状态
- 调了哪个工具
- 工具输入是什么
- 工具输出是什么
- 哪一步失败了
- 最终回复是什么

这一步会把项目从“能跑”提升到“可调试、可维护”。

### 3. Auth + Multi-user + Deployment

当 agent loop 跑通以后，下一步就应该让它更像真实产品：

- 登录
- 用户隔离
- 云数据库
- 部署
- 环境变量管理

这一步是“全栈”的关键分水岭。

很多人会卡在 demo 阶段，就是因为永远没有把“本地玩的聊天页面”升级成“真实在线服务”。

这些事项不是现在全做，而是作为后续产品演进 backlog。

- [ ] 新增 `AgentRun` 或 `ToolCall` 表，记录一次 agent 运行过程
- [ ] 给模型增加第一个工具：`get_time`

## 我建议你接下来最先学的 3 件事

### 1. Agent Loop 设计

这是最重要的。

不是继续学更多 React 组件技巧，也不是继续打磨聊天 UI，而是学：

- 模型什么时候该调用工具
- 工具结果怎么回灌给模型
- 服务端如何组织这轮运行

如果这一层打通了，你就真的开始往 AI agent 走了。

### 2. 数据建模

下一阶段你该开始思考的不只是 `Chat` 和 `Message`，还要有：

- `AgentRun`
- `ToolCall`
- `RunStatus`

因为 agent 产品的核心不是“消息”，而是“运行过程”。

### 3. 服务端可观察性

真正的全栈 AI 项目，调试能力比 UI 花活重要得多。

你接下来要习惯记录：

- 哪一步开始了
- 哪一步失败了
- 请求发给了谁
- 返回了什么错误

这会极大提升你做复杂 AI 产品的能力。

---

## 当前不建议优先做的事

- 继续反复打磨聊天 UI 细节
- 过早做复杂多 agent
- 一上来就做完整工作流平台
- 在还没有 tool calling 之前做太多产品花活

这些都不是没价值，而是优先级没那么高。

---

## 一句话版建议

当前项目的下一阶段，不是“把聊天做得更像聊天产品”，而是：

**把它升级成一个带最小工具调用、可追踪运行过程、未来能支持多用户的 AI agent 骨架。**

所以你接下来最该学的是：

1. `Tool Calling / Agent Loop`
2. `AgentRun / ToolCall` 这类数据建模
3. 服务端可观察性与错误处理

---

## 推荐执行顺序

1. 先做 `get_time` 工具
2. 补 `AgentRun` / `ToolCall` 数据表
3. 前端展示一次 agent 运行轨迹
4. 再做 `web_search`
5. 最后再进入登录、部署、多用户

如果后面你只选一个方向继续推进，就选这个：

**先把“单工具可追踪 Agent”做出来。**
